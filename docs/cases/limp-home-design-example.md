# limp-home design example (foundation)

本示例用于补齐 v6 在“受限运行策略（limp-home）”上的设计型说明缺口，聚焦一个范围可控场景：
**部分传感链路异常 + 单路功率受限** 时，系统进入 limp-home，仅保留最小可接受驱动能力与观测能力。

## scenario background

### 案例缺口清单
- v6 目前还没有一份明确偏“受限运行策略设计”的示例文档。
- 仓库虽已有 bring-up、incident、watchdog、report 等对象，但缺少一份说明“部分故障下如何保留有限能力”的设计说明。
- 需要先用一个边界清晰、可验证的 limp-home 场景，把进入条件、保留能力、禁止能力、风险边界与退出条件讲清楚。

### 场景定义
- 异常前提：主传感链路存在部分失效（如一组速度/姿态相关输入出现间歇丢帧、校验失败或偏差超阈）。
- 资源前提：动力系统退化为单路可用，持续输出能力受限，峰值扭矩与瞬态响应下降。
- 设计目标：不追求任务性能，只保证“可控、可观测、可安全退出”的最小运行能力。

## entry conditions

进入 limp-home 需要满足以下条件中的“故障触发 + 防误触确认”组合：
1. 传感链路故障触发：关键观测链路中至少 1 条进入不可信状态（示例判据：1 s 内连续 >=3 帧校验失败，或时间戳老化 >120 ms 持续 >=200 ms，或双源残差 >15% 持续 >=300 ms）。
2. 功率受限触发：功率管理报告仅剩单路可用，或双路中一路被保护逻辑锁定且状态持续 >=100 ms。
3. 防误触确认：上述状态需连续保持 >=400 ms，且在该窗口内未出现连续 >=200 ms 的“全量恢复可信”信号。
4. 进入互斥：若同时存在“不可控风险”信号（示例判据：制动不可用持续 >50 ms、转向反馈失联持续 >100 ms），则不进入 limp-home，直接进入更高等级失效安全策略（停车/切断输出）。

## retained capabilities

进入 limp-home 后仅保留以下最小能力：
- 低速受限驱动：限制最大车速、加速度和扭矩斜率（示例上限：车速 <=20 km/h、纵向加速度 <=0.8 m/s²、扭矩指令斜率 <=15%/s），仅允许缓慢移动到安全停靠点。
- 基本姿态/速度观测：使用冗余链路或降级估计，维持必要闭环稳定性监控（示例判据：观测刷新率 >=20 Hz、速度估计抖动 RMS <=0.5 km/h）。
- 基本告警与状态上报：持续输出“limp-home active、触发原因、当前约束参数”（示例时序：状态发布周期 <=100 ms）。
- 人机最小交互：仅保留人工确认“继续受限运行/请求退出到停机态”的输入能力，且任何人工输入都不得放宽既有安全约束。

## degraded / disabled capabilities

以下能力在 limp-home 期间必须降级或禁止：
- 禁止高动态动作：急加速、激烈变向、快速工况切换（示例判据：纵向加速度命令 >0.8 m/s²、横摆角速度命令 >8°/s、工况切换周期 <1 s 的请求一律拒绝）。
- 禁止高阶自动功能：依赖完整传感融合与全功率冗余的自动控制模式（示例判据：融合质量标志非 full 或功率冗余非 dual-ready 时禁止激活）。
- 禁止性能优先调度：任何会提升输出上限但削弱稳定裕度的策略（示例判据：目标车速上调 >20 km/h 上限或扭矩限幅放宽请求均拒绝）。
- 禁止故障掩盖：不得静默忽略传感异常，不得关闭关键健康监测（示例判据：任一健康监测通道失活 >100 ms 视为违规状态并触发告警）。

## risk boundary

本设计的边界不是“继续完成任务”，而是“在可解释风险内保持最小可控运行并安全退出”。

### risk-boundary-note 摘要
- 可接受风险：
  - 运行效率显著下降（示例判据：任务时长增加 <=2x）；
  - 响应变慢但仍可预测（示例判据：阶跃响应 10%~90% 上升时间 <=1.5 s 且无持续振荡）；
  - 仅在低速、低动态、短时窗口内移动（示例判据：车速 <=20 km/h、横向加速度 <=0.5 m/s²、连续运行窗口 <=10 min）。
- 不可接受风险：
  - 控制不稳定或指令不可预测（示例判据：同一输入在 1 s 内输出抖动峰峰值 >20%）；
  - 观测失真导致闭环失配且无法被监测识别（示例判据：关键观测残差 >20% 持续 >=300 ms 且未触发告警）；
  - 继续运行会扩大硬件损伤或引入人身安全风险（示例判据：热保护触发计数在 60 s 内 >=3 次或绝缘/过流告警持续 >100 ms）。
- 仍需验证点：
  - 传感异常间歇恢复时是否会引起模式抖动（示例判据：10 次注入中模式切换次数 <=1 次/事件）；
  - 单路功率热退化下的持续可控时长（示例判据：在限功率下可控运行 >=5 min）；
  - 边界条件下告警链路的可达性与时效性（示例判据：告警到达率 >=99%，端到端时延 <=200 ms）。

## exit conditions

limp-home 的退出遵循“先稳定、再切换”的收敛原则：
1. 恢复退出：传感链路与功率链路均恢复可信（示例判据：关键残差 <10%、时间戳老化 <80 ms、双路功率状态 dual-ready），且该状态连续保持 >=3 s 且切换计数不超过 1 次，方可退出到正常模式。
2. 停靠退出：到达预定义安全停靠条件后退出到安全停机态（示例判据：车速 <=3 km/h 持续 >=2 s、驱动扭矩请求为 0、驻车条件成立）。
3. 强制退出：若出现不可接受风险信号，立即终止 limp-home 并转入更高等级 failsafe（示例判据：触发后 <=100 ms 发出切换命令）。

### convergence-strategy-review 摘要
- 不允许“瞬时恢复即立刻回切”；必须满足稳定窗口与一致性检查。
- 回切前需完成约束参数清理与状态机一致性确认，避免模式跳变。
- 若恢复条件长期不满足，应优先执行停靠退出，而非延长受限运行时间。

## verification points

建议按“触发正确性、边界约束、退出收敛”三类进行最小验证：
- 触发正确性：
  - 注入传感链路间歇异常 + 单路功率受限，确认进入条件可重复命中（示例判据：20 次注入命中率 >=95%，进入时延 400~800 ms）；
  - 无故障与短抖动场景下不误入 limp-home（示例判据：30 min 稳态运行误触发 =0，<200 ms 抖动注入误触发率 <=1%）。
- 边界约束：
  - 驱动上限、扭矩斜率、速度上限在 limp-home 中始终生效（示例判据：全程无一次越限样本）；
  - 被禁能力不可被操作路径绕开（示例判据：所有高阶模式激活请求均返回 reject 且记录日志）。
- 退出收敛：
  - 恢复场景下满足稳定窗口后可平滑回切（示例判据：回切后 5 s 内无二次跌回 limp-home）；
  - 不可接受风险出现时可及时升级到 failsafe（示例判据：升级命令发出时延 <=100 ms）。

### failsafe-check-matrix 摘要
- Case A：仅传感异常、功率正常 -> 不进入本案例 limp-home，按传感降级策略处理（该策略为本案例外依赖，此处不展开）。
- Case B：传感异常 + 单路功率受限 -> 进入 limp-home，执行最小能力约束。
- Case C：叠加不可接受风险信号 -> 立即退出 limp-home，转入 failsafe。
