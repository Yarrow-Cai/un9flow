# power board bring-up example

## case overview
本案例聚焦于功率板首次上电后的早期 bring-up：预充流程显示完成，但 PWM 输出建立不稳定（表现为可短暂建立后抖动掉线，且在重复使能时复现概率升高）。

案例缺口清单（为何需要此文档）：
- v6 仍缺少一份明显偏功率板 bring-up 的端到端示例，难以直接复用到功率板场景。
- 当前仓库虽已有 bringup-path 与链路类 bring-up 模板，但缺少“safety baseline → control establishment → escalation decision”串联到同一案例中的落地样例。
- 需要先用范围可控的功率板问题，证明 bringup-path 在功率板故障定位中的实际用法与边界。

## safety baseline
在进入控制建立前，先锁定“可重复、可回退、不伤板”的安全基线，避免把不确定问题扩大：
- 上电路径保持限流与受控上电时序，确认预充完成判据可稳定复现。
- PWM 使能前确认关断链路有效（急停、驱动失能、主功率关断路径可独立生效）。
- 仅在最小负载/替代负载下做首轮控制建立，避免在未知状态直接进入高能量工况。

### bringup-baseline 摘要
- 已确认：母线电压上升曲线与预充结束阈值一致，未出现明显过冲。
- 已确认：驱动电源与逻辑电源时序满足最小上电窗口。
- 约束条件：当前记录仅覆盖低风险测试窗，不代表满载稳定性结论。

## bring-up path usage
采用 bringup-path 的“先链路、后控制、再性能”路径，避免直接把问题归因到单一模块：
1. 先验证控制链路到执行链路的可达性（命令可下发、状态可回读、保护状态可见）。
2. 再做 PWM 最小占空比建立测试，观察建立瞬间与保护位变化。
3. 最后才尝试扩大占空比/延长维持时间，确认是否与能量水平相关。

- 最小可复现实验 / 判定门槛骨架（低风险窗口）：
  - 前置条件：safety baseline 三项均已确认；仅最小负载；记录窗口与测试批次已编号。
  - 操作顺序：按“使能 PWM 2 s → 关断 3 s”执行 10 轮（R01-R10），轮间不改参数；随后仅调整 1 个控制参数复测 3 轮（R11-R13）。
  - 观测点：PWM 建立耗时、建立后 500 ms 内是否掉线、软保护状态位翻转次数、命令/状态链路是否丢样。
  - 判定门槛：10 轮内若 ≥3 轮出现“500 ms 内掉线”或出现连续 2 轮掉线，判定为“不稳定”；若 R11-R13 仍出现 ≥1 轮掉线，满足 bring-up -> escalation 的升级门槛。

### link-qualification-log 摘要
- 命令下发链路：可达，时延在可接受范围内。
- 状态回读链路：可达，但在 PWM 抖动阶段出现状态位短时翻转。
- 保护信号观测：未见硬故障锁死，但出现间歇性软保护触发痕迹。
- 追溯锚点：本摘要统一使用「窗口 W1 / 批次 B01 / 回合 Rxx」标识，可直接回指单轮原始观测。

## control establishment
在安全基线和链路可达性确认后，进入控制建立：
- 现象：PWM 使能后可短时建立，随后频率/占空比输出抖动并退回非稳定状态。
- 复现特征：冷启动前两次更易建立，连续重试后不稳定概率上升。
- 关联线索：抖动时段伴随软保护相关状态位变化，但未触发不可恢复保护。
- 阶段 gate（control establishment -> design review / incident）：在同一窗口同一批次内，若 5 轮中出现 ≥2 轮“建立后 500 ms 内掉线”且至少 1 轮伴随软保护位翻转，则转 design review；若出现连续 2 轮掉线且伴随不可恢复保护/需人工断电恢复，则直接转 incident。

### initial-diagnosis-conclusion 摘要
- 初步结论：问题更接近“控制建立窗口与保护阈值/滤波参数耦合”而非“链路完全中断”。
- 证据强度：中等（样本来自低风险窗口 W1、批次 B01、回合 R01-R13，可复现但数量有限）。
- 暂不支持结论：尚不能直接判定为功率器件失效或主控算法根因。

## escalation decision
当“可建立但不稳定”在受控条件下持续复现，应升级到跨域会诊，而不是在单一参数上盲目扫描：
- 升级触发条件（bring-up -> escalation）：
  - 同一安全窗口（如 W1）10 轮内出现 ≥3 轮“建立后 500 ms 内掉线”，或连续 2 轮掉线；
  - 软保护痕迹在 ≥2 轮与 PWM 抖动时间对齐；
  - 单参数调整复测 3 轮后仍出现 ≥1 轮掉线（无显著改善）。
- 升级目标：
  - 硬件侧核查采样链路完整性、保护阈值离散性、门极驱动边界；
  - 控制侧核查建立阶段参数切换逻辑、保护滤波时间常数、状态机跃迁条件。

### risk-boundary-note 摘要
- 本案例结论边界：用于说明 bringup-path 在功率板问题中的落地流程，不构成“已彻底修复”的证明。
- 禁止外推：不得将当前低负载结论直接外推到额定功率与长期运行场景。

## final outcome
本案例的价值是建立“从安全基线到升级判断”的可执行路径，而非一次性给出完美根因闭环。

- 已确认项：
  - 预充完成判据与基础上电时序在当前窗口内可复现；
  - 控制与状态链路并未完全中断；
  - PWM 不稳定与软保护痕迹存在时间关联。
- 未确认项：
  - 保护触发的主因是硬件采样偏差、参数边界设置，还是控制建立策略时序；
  - 该现象在更高负载/更长运行时长下的演化规律。
- 下一步建议：
  - 补充同步波形与状态机日志的对齐采集；
  - 进行阈值与滤波参数的受控矩阵试验；
  - 若升级后仍无收敛，再进入器件级排查与替换对比。
